# 第二章接口取数与清洗契约示范

## 目的

本文件不是报告写作指南，而是给接口取数层和本地清洗层使用的“取数契约”示范。

第二章「利润概况」需要从接口取出一张利润瀑布表，清洗成稳定结构后，再交给第二章报告生成器。

一句话链路：

```text
客户范本第二章利润表
-> 反推需要的利润科目和时间维度
-> 接口 MOUDLE=2 返回这些指标行
-> 本地按 指标路径 + 日期类型 抽数
-> 生成 chapter2_final_data.json
-> 生成 chapter2_final_report.md/html/pdf
```

## 一期做法对照

一期不是向接口逐个传字段名取数，而是：

1. 按模块请求接口，例如 `MOUDLE=2`。
2. 接口返回该模块完整 `data.章节数据`。
3. 本地在 `postprocess_data.py` 里按 `指标路径`、`指标名称` 筛选需要的指标。
4. 清洗成章节生成器需要的数据结构。

二期沿用这个方式，但要把“需要筛哪些指标”显式沉淀成契约表，避免清洗规则散落在代码里。

## 接口必须返回的指标

第二章需要 11 个利润科目，每个科目需要 3 个时间维度。

时间维度：

| 日期类型 | 报告列名 | 说明 |
| --- | --- | --- |
| `月` | 报告月，例如 6月 | 当月值 |
| `季` | 本季度累计 | 本季度累计值 |
| `年` | 1-N月累计 | 年初至报告月累计值 |

利润科目：

| 序号 | 指标名称 | 指标路径 | 必须返回日期类型 | 取数值 |
| ---: | --- | --- | --- | --- |
| 1 | 营业收入（不含税） | 二、利润概况-营业收入（不含税） | 月、季、年 | 指标数据.实际值 |
| 2 | 毛利率 | 二、利润概况-毛利率 | 月、季、年 | 指标数据.实际值 |
| 3 | 毛利额 | 二、利润概况-毛利额 | 月、季、年 | 指标数据.实际值 |
| 4 | 个人人工费用 | 二、利润概况-个人人工费用 | 月、季、年 | 指标数据.实际值 |
| 5 | 其中：基本薪酬+提成奖金+年终奖 | 二、利润概况-其中：基本薪酬+提成奖金+年终奖 | 月、季、年 | 指标数据.实际值 |
| 6 | 差旅费 | 二、利润概况-差旅费 | 月、季、年 | 指标数据.实际值 |
| 7 | 减值损失 | 二、利润概况-减值损失 | 月、季、年 | 指标数据.实际值 |
| 8 | 财务费用 | 二、利润概况-财务费用 | 月、季、年 | 指标数据.实际值 |
| 9 | 样板物料 | 二、利润概况-样板物料 | 月、季、年 | 指标数据.实际值 |
| 10 | 其他各类费用 | 二、利润概况-其他各类费用 | 月、季、年 | 指标数据.实际值 |
| 11 | 分摊前利润 | 二、利润概况-分摊前利润 | 月、季、年 | 指标数据.实际值 |

每条指标行的 `指标数据` 至少需要：

| 字段 | 是否必需 | 用途 |
| --- | --- | --- |
| `实际值` | 必需 | 报告表格主值 |
| `日期类型` | 必需 | 判断月/季/年列 |
| `单位` | 必需 | 展示和单位校验 |
| `目标值` | 可选 | 当前第二章表格不用；后续若做目标对比可用 |
| `同期数` | 可选 | 当前第二章若做同比趋势需要 |
| `达成率` | 可选 | 当前第二章表格不用 |
| `扣分值` | 可选 | 当前第二章表格不用 |

## 字段映射

清洗字段按以下规则生成。

| 清洗字段 | 来源指标路径 | 日期类型 | 来源值 | 单位规则 |
| --- | --- | --- | --- | --- |
| `profit_table.revenue.month` | 二、利润概况-营业收入（不含税） | 月 | 实际值 | 万元 |
| `profit_table.revenue.quarter` | 二、利润概况-营业收入（不含税） | 季 | 实际值 | 万元 |
| `profit_table.revenue.year` | 二、利润概况-营业收入（不含税） | 年 | 实际值 | 万元 |
| `profit_table.gross_margin_rate.month` | 二、利润概况-毛利率 | 月 | 实际值 | 转为 % |
| `profit_table.gross_margin_rate.quarter` | 二、利润概况-毛利率 | 季 | 实际值 | 转为 % |
| `profit_table.gross_margin_rate.year` | 二、利润概况-毛利率 | 年 | 实际值 | 转为 % |
| `profit_table.gross_margin_amount.*` | 二、利润概况-毛利额 | 月/季/年 | 实际值 | 万元 |
| `profit_table.labor_cost.*` | 二、利润概况-个人人工费用 | 月/季/年 | 实际值 | 万元 |
| `profit_table.salary_bonus_cost.*` | 二、利润概况-其中：基本薪酬+提成奖金+年终奖 | 月/季/年 | 实际值 | 万元 |
| `profit_table.travel_cost.*` | 二、利润概况-差旅费 | 月/季/年 | 实际值 | 万元 |
| `profit_table.impairment_loss.*` | 二、利润概况-减值损失 | 月/季/年 | 实际值 | 万元 |
| `profit_table.financial_expense.*` | 二、利润概况-财务费用 | 月/季/年 | 实际值 | 万元 |
| `profit_table.sample_material.*` | 二、利润概况-样板物料 | 月/季/年 | 实际值 | 万元 |
| `profit_table.other_expense.*` | 二、利润概况-其他各类费用 | 月/季/年 | 实际值 | 万元 |
| `profit_table.pre_allocation_profit.*` | 二、利润概况-分摊前利润 | 月/季/年 | 实际值 | 万元 |

`*` 表示同时生成 `month`、`quarter`、`year` 三个字段。

## 派生计算

第二章报告分析还需要从基础字段派生一些判断，不要求接口直接给。

| 派生字段 | 计算方式 | 用途 |
| --- | --- | --- |
| `expense_total.{dim}` | 个人人工费用 + 差旅费 + 减值损失 + 财务费用 + 样板物料 + 其他各类费用 | 费用分析 |
| `expense_income_ratio.{dim}` | 费用合计 / 营业收入 | 判断费用是否偏高 |
| `largest_expense_items.{dim}` | 费用项按金额降序排序 | 找最大费用项 |
| `profit_status.{dim}` | 分摊前利润 >= 0 为盈利，否则亏损 | 利润表现说明 |

注意：

1. `其中：基本薪酬+提成奖金+年终奖` 是个人人工费用子项，不重复计入费用合计。
2. 营业收入为 0 或缺失时，不计算费用收入比，展示 `—`。
3. 同期数缺失或为 0 时，不计算同比增长。

## 缺失处理

| 情况 | 处理 |
| --- | --- |
| 某科目某日期类型缺失 | 表格填 `—`，warnings 记录缺失 |
| `实际值` 为空或无法转数字 | 表格填 `—`，warnings 记录无法转换 |
| `毛利率` 单位不是 `%` | 若数值小于等于 1，按比例转百分比并 warning；否则按原值百分比展示并 warning |
| 同一指标路径 + 日期类型出现多条 | 保留第一条，warnings 记录重复 |
| `目标值` 或 `同期数` 为 0 | 不自动解释为真实目标/同期，除非接口确认 |

## 给第二章生成器的数据

第二章生成器不应该直接拿接口 raw rows 做报告，而应该拿这个结构：

```json
{
  "profit_table": {
    "revenue": {
      "month": {"value": 0.215, "display": "0.2万元"},
      "quarter": {"value": 0.999, "display": "1.0万元"},
      "year": {"value": 7.041, "display": "7.0万元"}
    },
    "gross_margin_rate": {
      "month": {"value": 0.242, "display": "24.2%"},
      "quarter": {"value": 0.214, "display": "21.4%"},
      "year": {"value": 0.303, "display": "30.3%"}
    }
  },
  "derived": {
    "expense_total": {
      "year": {"value": 74.468, "display": "74.5万元"}
    },
    "largest_expense_items": [
      {"subject_key": "other_expense", "subject_name": "其他各类费用", "value": 51.789, "unit": "万元"}
    ]
  },
  "warnings": [
    "毛利率接口单位为万元，已按比例值转为百分比展示。"
  ]
}
```

这样第二章报告生成器只负责写报告，不再负责猜接口字段。

## 第二章完成标准

第二章清洗契约完成后，应该满足：

1. `MOUDLE=2` raw JSON 可以完整命中 11 个科目 × 3 个日期类型。
2. 输出 `chapter2_final_data.json`。
3. 毛利率正确展示为百分比，不再出现 `0.24万元`。
4. 缺失字段不会导致生成失败，报告表格用 `—` 占位。
5. warnings 只写入 final data，不写入客户报告正文。
