# 第一章接口字段映射表（字段级契约版）

章节：一、绩效得分与预警  
接口模块：`MOUDLE = "1"`  
适用对象：区域经理单人报告  
映射目标：报告中每一个关键展示值，都能明确追溯到接口的某一条 `章节数据` 记录和某一个字段。

## 1. 映射约定

### 1.1 接口记录结构

第一章所有业务指标来自：

```text
data.章节数据[]
```

每条记录结构：

```text
指标名称
指标路径
指标数据.实际值
指标数据.目标值
指标数据.同期数
指标数据.扣分值
指标数据.达成率
指标数据.权重分数
指标数据.单位
指标数据.分部排名
指标数据.省区排名
指标数据.部门排名
指标数据.日期类型
```

### 1.2 匹配规则说明

| 字段 | 含义 |
| --- | --- |
| `field_id` | 报告字段唯一 ID，代码取数和 diagnostics 都使用该 ID |
| `report_position` | 该字段在报告中的位置 |
| `source_module` | 接口模块，第一章固定为 `1` |
| `record_match` | 从 `data.章节数据[]` 中定位记录的条件 |
| `value_path` | 从命中的记录里取哪个字段 |
| `unit_path` | 单位字段，可为空 |
| `format` | 展示格式 |
| `required` | 是否必需 |
| `fallback` | 主取数失败时的备用规则 |
| `missing_behavior` | 缺失时如何处理 |
| `validation` | 取到值后的校验规则 |

### 1.3 业务名词约定

| 接口字段 | 报告展示 |
| --- | --- |
| `指标数据.省区排名` | 省区内排名 |
| `指标数据.部门排名` | 事业部内排名 |
| `指标数据.分部排名` | 分部排名，当前第一章总览表暂不展示 |

说明：接口中没有直接叫“事业部排名”的字段，当前用 `部门排名` 映射为报告里的“事业部内排名”。

| report_field | 当前状态 | 建议 |
| --- | --- | --- |
| `chapter1.rank_table.province_total` | 第一章接口未提供 | 从人员名单接口按当前人员省区统计，或由章节接口新增 |

## 7. 格式化规则

| format | 规则 | 示例 |
| --- | --- | --- |
| `integer` | 转整数，不带单位 | `24` |
| `amount_3_trim_unit` | 最多保留 3 位小数，去掉多余 0，追加单位 | `145.611万元`、`44.987万元` |
| `amount_2_wan` | 保留 2 位小数，单位为万 | `0.14万` |
| `score_1_unit` | 保留 1 位小数，追加单位 | `55.7分` |
| `percent_from_decimal` | 小数转百分比 | `0.557 -> 55.7%` |
| `percent` | 若接口单位为 `%`，直接按百分数格式展示 | `85.1%` |
| `text` | 原样展示 | `刘晨` |
| `period_yyyymm` | `202606 -> 2026年1-6月` | `2026年1-6月` |

## 8. 缺失与校验规则

### 8.1 必需字段缺失

必需字段缺失时：

```text
报告显示 —
diagnostics 记录 field_id、record_match、value_path、缺失原因
```

### 8.2 命中多条

命中多条时：

1. 若映射表定义了排序或筛选规则，按规则选择。
2. 若无法唯一选择，报告显示 `—`。
3. diagnostics 记录所有候选记录摘要。

### 8.3 类型错误

例如排名字段不是数字：

```text
报告显示 —
diagnostics 记录 value_path 类型错误
```

### 8.4 值为 0 的处理

排名类字段中 `0` 视为无有效排名。

```text
省区排名 = 0 -> 缺失
部门排名 = 0 -> 缺失
```

## 10. 建议的机器可读版本

后续代码不应直接从 Markdown 表格读取规则，而应维护一份机器可读映射，例如 YAML/JSON。

示例：

```yaml
chapter1.rank_table.sales_province_rank:
  source_module: 1
  report_position: 总览表 / 省区内排名 / 销量
  record_match:
    指标名称: 销量排名-省区
  value_path: 指标数据.省区排名
  format: integer
  required: true
  missing_behavior: display_dash_and_diagnostic
  validation:
    type: integer
    gt: 0
```

这份 Markdown 表用于业务审查；通过后应转成 YAML/JSON 并由代码执行。
