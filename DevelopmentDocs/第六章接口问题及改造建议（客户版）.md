# 第六章《费用分析》接口问题及改造建议

## 一、接口信息

请求地址：

```text
POST /test/skshu-bi-api/biapitoxt/getEmployeeIndexAiCcq
```

请求参数：

```json
{
  "ZEMPLOYEE": "06427",
  "CALMONTH": "202606",
  "MOUDLE": "6"
}
```

本次接口请求成功，返回第六章《费用分析》数据共 43 条。

## 二、当前接口数据示例

### 1. 差旅费

报告月返回了 5 条相同指标名称、相同指标路径、相同单位的记录：

```json
{
  "指标名称": "差旅费",
  "指标路径": "六、费用分析-差旅费",
  "指标数据": {
    "实际值": "7115.500 / 418.000 / 1337.500 / 3840.000 / 1520.000",
    "单位": "元",
    "日期类型": "月"
  }
}
```

接口没有说明上述 5 个数值分别对应：差旅费总额、交通费、住宿费、车辆费或其他费用。

### 2. 出差效率

出差天数可以唯一识别：

```json
{
  "指标名称": "差旅费",
  "指标路径": "六、费用分析-差旅费",
  "指标数据": {
    "实际值": "49.000",
    "单位": "天",
    "日期类型": "月"
  }
}
```

每天花费金额返回 2 条相同标识的记录：

```json
{
  "指标名称": "每天花费金额",
  "指标路径": "六、费用分析-差旅费-每天花费金额",
  "指标数据": {
    "实际值": "27.300 / 31.020",
    "单位": "元",
    "日期类型": "月"
  }
}
```

接口没有说明两条记录分别对应哪类日均费用，也没有提供日均总费用的唯一标识。

### 3. 样板样漆费用

报告月总额可以唯一识别：

```json
{
  "指标名称": "样板样漆费用",
  "指标路径": "六、费用分析-样板样漆费用",
  "指标数据": {
    "实际值": "4398.500",
    "同期数": 4172.47,
    "单位": "元",
    "日期类型": "月"
  }
}
```

产品明细返回 7 条相同指标名称、相同路径的记录，金额分别为：

```text
894.400、199.800、429.000、516.000、344.000、344.000、1429.200 元
```

这些记录没有产品名称或产品编码，无法确定每个金额对应的产品。

## 三、当前问题及影响

| 当前问题 | 报告影响 |
| --- | --- |
| 同名、同路径记录缺少费用类别 | 无法区分差旅费总额、交通费、住宿费、车辆费和其他费用 |
| 日均费用记录缺少类别 | 无法展示日均总费用及各类日均费用 |
| 样板样漆明细缺少产品名称/编码 | 无法生成费用金额前三的产品 |
| 接口未提供同比增长率、同比差额字段 | 无法直接展示同比变化 |
| 年、月数据混合返回 | 必须依赖 `日期类型` 区分统计口径 |

为保证报告数字准确，目前只能展示可唯一定位的 `49.000天` 和 `4398.500元`；其余位置不能按数组顺序或金额大小推测，因此显示“待补充”。

## 四、建议客户修改内容

### 1. 为每条指标提供明确且唯一的业务标识

建议将差旅费相关记录改为：

```json
[
  {
    "指标编码": "TRAVEL_TOTAL",
    "指标名称": "差旅费总额",
    "指标路径": "六、费用分析-差旅费-差旅费总额",
    "指标数据": {"实际值": "<总额>", "单位": "元", "日期类型": "月"}
  },
  {
    "指标编码": "TRAVEL_TRANSPORT",
    "指标名称": "交通费",
    "指标路径": "六、费用分析-差旅费-交通费",
    "指标数据": {"实际值": "<交通费金额>", "单位": "元", "日期类型": "月"}
  },
  {
    "指标编码": "TRAVEL_HOTEL",
    "指标名称": "住宿费",
    "指标路径": "六、费用分析-差旅费-住宿费",
    "指标数据": {"实际值": "<住宿费金额>", "单位": "元", "日期类型": "月"}
  },
  {
    "指标编码": "TRAVEL_VEHICLE",
    "指标名称": "车辆费",
    "指标路径": "六、费用分析-差旅费-车辆费",
    "指标数据": {"实际值": "<车辆费金额>", "单位": "元", "日期类型": "月"}
  },
  {
    "指标编码": "TRAVEL_OTHER",
    "指标名称": "其他费用",
    "指标路径": "六、费用分析-差旅费-其他费用",
    "指标数据": {"实际值": "<其他费用金额>", "单位": "元", "日期类型": "月"}
  }
]
```

### 2. 明确出差效率指标

建议分别提供：

- 出差天数
- 日均花费总额
- 日均交通费
- 日均住宿费
- 日均车辆费
- 日均其他费用

每项使用独立的 `指标编码`、`指标名称` 和 `指标路径`。

### 3. 为样板样漆明细增加产品身份字段

建议结构：

```json
{
  "指标编码": "SAMPLE_PAINT_PRODUCT_EXPENSE",
  "指标名称": "样板样漆产品费用",
  "指标路径": "六、费用分析-样板样漆费用-产品费用",
  "产品编码": "<产品编码>",
  "产品名称": "<产品名称>",
  "指标数据": {
    "实际值": "<产品费用>",
    "单位": "元",
    "日期类型": "月"
  }
}
```

### 4. 直接提供同比字段

如报告需要展示同比，建议在 `指标数据` 中增加：

```json
{
  "同期数": "<同期值>",
  "同比差额": "<实际值减同期值>",
  "同比增长率": "<同比增长率>"
}
```

请同时明确百分比的传输口径，例如 `5.42` 表示 `5.42%`，避免小数和百分数口径混用。

## 五、验收标准

接口调整后，应满足以下条件：

1. 每个报告字段可以通过明确条件唯一命中一条记录。
2. 不依赖数组顺序或金额大小判断业务含义。
3. 差旅费总额及四类明细均有独立标识。
4. 日均费用总额及四类明细均有独立标识。
5. 样板样漆明细包含产品编码和产品名称。
6. 同比字段及百分比口径明确。
7. 接口数字保持原始精度。

完成以上调整后，第六章报告即可稳定生成全部费用分析内容，无需人工判断或补录。
